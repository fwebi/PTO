# User Manual for PTO Tracking System 

 

## Installation Guide 

Download ZIP file and import project onto VSCode 

Install Node.js here  https://nodejs.org/en and create Environment PATH  

Once imported onto VSCode inspect Backend folder for SQL database file 

If the file is there, create a terminal in VSCode and change directory to /backend 

Once in the directory run command “npm install” to install node.js dependencies 

Once user has installed dependencies the backend is ready to start 

Create another terminal and change directory to /frontend and repeat steps 4 and 5 for the frontend directory 

Once both terminals have installed dependencies for frontend and backend you can begin. 

In the backend terminal run command “npm run dev” to start the server 

Once the backend server is running, switch to the frontend terminal and run command “npm start” to start the application.

Register, Login

## Employee Guide 

1. Once logged the employee dashboard will display Employee info 

2. Employee can use dropdown for date selectors to request days off 

3. Can change PTO type with drop down menu 

4. Will show number of days left for PTO/Sick usage	 

## Manager Guide 

1. Once logged the manager will be shown all employee PTO requests 

2. Requests can be approved or denied by management 

## Human Resources Guide 

1. Once logged HR dashboard will have options to update PTO policy 

2. Report button redirects to a new page generating a table report of all users 	and 	available PTO they have 

3. Create new user link redirects to form page where HR can create a new user by 	entering basic information like name, email and password.  

4. Can also assign roles during user creation in case of new management, 		employee or Human resource
